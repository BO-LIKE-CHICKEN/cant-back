# 대칭으로 맞추기

## 코드는 유기체처럼 성장한다

코드는 시간이 지남에 따라 새로운 기능과 요구 사항에 따라 성장하고 변화합니다. 이 과정에서 코드의 대칭성을 유지하는 것은 유지보수성과 가독성을 높이는 데 중요합니다. 대칭적인 코드 구조는 읽기 쉽고, 확장하기 쉬우며, 버그를 줄이는 데 도움이 됩니다.

<Callout>
  대칭적인 코드란 한 가지 방식을 선택해서 각기 다른 방식으로 작성한 동일한
  동작의 코드를 하나의 작성 방식으로 작성하는 것을 의미합니다.
</Callout>

## 예시

사용자의 나이에 따라 다른 메시지를 표시하는 기능을 구현

### 1. if...else 문을 사용한 구현

```ts
function getUserCategory(age) {
  if (age >= 18) {
    return "성인입니다.";
  } else if (age >= 13) {
    return "청소년입니다.";
  } else {
    return "어린이입니다.";
  }
}

console.log(getUserCategory(20)); // 출력: 성인입니다.
```

### 2. 삼항 연산자를 사용한 구현

```ts
function getUserCategory(age) {
  return age >= 18
    ? "성인입니다."
    : age >= 13
    ? "청소년입니다."
    : "어린이입니다.";
}

console.log(getUserCategory(20)); // 출력: 성인입니다.
```

### 3. switch 문을 사용한 구현

```ts
function getUserCategory(age) {
  switch (true) {
    case age >= 18:
      return "성인입니다.";
    case age >= 13:
      return "청소년입니다.";
    default:
      return "어린이입니다.";
  }
}

console.log(getUserCategory(20)); // 출력: 성인입니다.
```

같은 동작을 다양한 방식으로 구현할 수 있지만, 코드 스타일의 통일성은 코드의 품질과 유지보수성에 큰 영향을 미칩니다. 팀 내에서 합의된 스타일 가이드를 따르고, 상황에 맞는 적절한 조건문을 사용하여 일관된 코드를 작성해야 합니다.

- **가독성 향상**: 일관된 스타일은 코드를 읽기 쉽게 만듭니다.
- **유지보수성 향상**: 코드 수정 및 기능 추가 시 혼란을 줄입니다.
- **팀 생산성 증가**: 모든 팀원이 같은 규칙을 따르면 협업이 원활해집니다.
