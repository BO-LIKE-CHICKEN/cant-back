# 설명하는 주석

## 코드에서 명확하지 않은 내용만 골라 적으세요

```ts
// 'antd에서는 명시적 `undefined` 할당이 필요하므로 `undefined`를 할당해주는 로직이 존재합니다'
```

## 요점은 다른 사람의 관점에서 생각하고 예상되는 질문을 선제적으로 언급하려고 노력하는 것

- 가령, 코드리뷰에서 질문을 받았으나 특별한 사정에 의해 추가된 맥락을 설명하고 통과된 경우에는 설명했던 맥락을 주석으로 남기는 것은 좋을 것 같습니다.

## 코드의 결함을 발견했다면, 그 즉시 해당 위치에 주석을 달아야 합니다

- TODO로 꼭 남겨두기

## 정리

- 코드에서 명확하지 않은 내용만 골라 적으세요
- 다른 사람의 관점에서 생각하고 예상되는 질문을 선제적으로 적으세요
- 코드의 결함을 발견했다면, 그 즉시 해당 위치에 주석을 달아야 합니다 (지라티켓이 저의 경우에는 더 적절하게 작용했던 것 같습니다)

> `tsdoc`을 적극 활용하는 것이 가장 와닿습니다
