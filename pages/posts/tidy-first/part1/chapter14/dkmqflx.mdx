## 14. 설명하는 주석

> 코드에서 명확하지 않은 내용만 골라적으세요. 자신이 이 코드를 처음 읽는 사람이라고 가정해보세요.

> 코드의 결함을 발견했다면. 그 즉시 해당 위치에 주석을 달아야 합니다.
> …
> 결국 이를 제거하는 방법을 배워야 하겠지만, 그 때까지는 주석을 달아서 결합도 문제를 미리 지적해 두는 것이, 모래 속에 묻듯 그야 두는 것보다 훨씬 나을 것입니다.

- 주석을 달 때에도 컨벤션을 정할 필요가 있다.

- TSDoc

  ```jsx
  /**
   * ## 복잡한 아이템
   * ### 복잡한 대한 타입 정의.
   */
  export type ComplexItem = {
    /** Id */
    id: number
    /** 이름 */
    name: string
    /** 가격 */
    price: number
  }
  ```

<br />

- Todo Tree

  ```jsx
  // TODO: 리팩토링
  ```

- “주석이 필요 없는 코드를 작성해야 한다”는 말은 현실과 동떨어진 소리라고 생각

- 내가 작성하지 않은 오래전에 작성된 코드를 마주치거나 레거시한 기존 코드에 새로운 코드를 추가해야하는 경우가 있는데, 이 때 그 상황에 맞는 주석을 작성해 준다면 가독성을 높이고 추후에 리팩토링을 할 때에도 도움을 받을 수 있기 때문
