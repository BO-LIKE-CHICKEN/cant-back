# 설명하는 상수

## 상징적인 상수를 만드세요. 리터럴 상수로 사용된 곳은 상징적인 상수로 바꿉니다.

### 사례

이 책에서 계속 잊지말아야 하는 것은 **이후에 수반되는 리팩터링이나 기능 추가**등을 수월하게 하기 위해 코드를 정리한다는 것 입니다.
그리고 리터럴 상수는 만드는 것은 그 목적을 달성하기 위해 꼭 진행되어야 합니다.

리터럴 상수를 상징적인 상수로 변경하면 코드의 가독성과 유지보수성이 향상됩니다. 이를 통해 값이 변경될 때 코드 곳곳을 수정할 필요가 없고, 상수의 의미를 명확하게 표현할 수 있습니다.

#### AS-IS

```ts
function calculateTaxiFare(isNight: boolean): number {
  if (isNight) {
    return 6_240;
  }

  return 4_800;
}
```

#### TO-BE

```ts
const 택시_기본_요금 = 4_800;
const 택시_야간할증_기본_요금 = 6_240;

function calculateTaxiFare(isNight: boolean): number {
  if (isNight) {
    return 택시_야간할증_기본_요금;
  }

  return 택시_기본_요금;
}
```

### 변경으로 인한 개선점

#### 가독성 향상:

숫자 `4_800`과 `6_240`이 각각 `택시_기본_요금`와 `택시_야간할증_기본_요금`로 바뀌면서 코드의 의도를 명확하게 알 수 있습니다. 상수명을 통해 이 값들이 택시 요금이라는 것을 쉽게 알 수 있습니다.

#### 유지보수성:

택시 요금이 바뀌어야 할 때, 상수만 수정하면 되므로 여러 곳의 코드를 변경할 필요가 없습니다.

#### 의미 전달:

상수명을 통해 해당 값의 의미를 알 수 있으므로, 코드를 읽는 사람은 이 값이 택시 요금과 관련이 있다는 것을 즉시 파악할 수 있습니다.

## BONUS

### 이런 상수화는 크게 도움이 되지 않습니다

```ts
const ONE = 1;
```

### 상수를 정리할때는 몇 가지를 고려하세요.

- 한 번에 바뀌어야 하거나 함께 이해해야 하는 상수들을 한곳에 모아둡니다.
- 다른 이유로 묶인 변수들을 분리하는 후속 작업을 해야합니다.
