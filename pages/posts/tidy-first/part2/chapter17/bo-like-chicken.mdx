# 연쇄적인 정리

이번 챕터는 part1의 반복느낌이라 간단하게 정리합니다.

## 코드를 계속 정리하고 싶은 충동을 관리하는 것도 코드 정리의 핵심 기술입니다.

책에서 권하는 것은 아주 작은 단계로 나누어 코드를 정리하는 방식을 고수하면서 실험하는 것

### 어떤 정리 방식들이 있을까?

### 보호구문

보호구문을 넣는다 -> 도우미나 상수로 추출할 수 있는 조건이 보인다

### 안 쓰는 코드

사용하지 않는 코드를 삭제하면 순서와 응집도가 좋아진다

### 대칭으로 맞추기

유사한 코드들이 묶여진 순서대로 읽을 수 있다

### 새로운 인터페이스로 기존 루틴 부르기

새 인터페이스를 사용하기 위해 기존의 루틴을 호환하도록 변환해 주는 작업을 거쳐야 한다

### 읽는 순서

읽는 순서를 정리한다 -> 대칭으로 맞출 기회가 생긴다

### 응집도를 높이는 배치

응집도를 있게 묶인 코드는 하위 요소로 추출할 후보가 된다

### 설명하는 변수

변수 이름으로 알 수 있는 설명 덕분에 불필요한 주석을 삭제할 수 있다

### 설명하는 상수

한 번에 바뀌어야 하는 상수를 모아서 묶어 놓으면 바꾸기 쉽다

### 명시적인 매개변수

파라미터를 여러 개 사용하는 것보다는 객체의 묶음으로 사용하는 것이 좋다 정도로 넘어가겠습니다

### 비슷한 코드끼리

코드 덩어리에 설명하는 주석을 붙이거나 도우미로 추출 가능하다

### 도우미 추출

도우미를 추출하면 앞서 설명한 도구들을 적용할 곳이 보인다

### 하나의 더미

일단 모아 놓으면 어떻게 묶고 정리해야할지 보인다

### 설명하는 주석

가능하다면 주석에 있는 정보를 코드로 옮깁니다

### 불필요한 주석 지우기

변경 중에서도 코드를 이해하는 일에 가장 비용이 많이 들어갑니다.
